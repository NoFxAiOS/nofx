# NOFX 架构/开发决策记录

## 2026-03-21 — 决定先做项目接管与文档治理，而非直接功能开发

### 背景
用户目标是推进加密货币 AI 交易系统的深度开发优化，并特别强调：
- 熟悉完整架构
- 读懂各模块
- 形成中文结构文档与注释
- 优化记忆与 token 使用
- 保证后续新增功能有良好衔接

### 决策
先完成“认知地基 + 治理文件 + 测试基线”，再进入正式功能开发。

### 原因
1. 交易系统属于高风险系统，直接改功能容易引入不可见错误
2. 没有统一架构文档，后续会话衔接成本高
3. 收益优化必须建立在稳定性和一致性之上
4. 文档化与决策记录可以显著降低 token 消耗和上下文丢失风险

### 影响
- 短期内进度看起来偏“治理型”而非“功能型”
- 中期会明显提高后续开发速度与质量
- 为多 agent / 多子任务协作提供统一基础
