## 3. 已做的关键决策

1. **先做接管与治理，不先做激进功能开发**
   - 交易系统风险高，必须先建立正确性与稳定性基线

2. **先清外部问题和一致性问题，再谈收益优化**
   - 没有一致性，收益评估不可信

3. **性能优化只做低风险边界改造**
   - 采用懒加载、拆 chunk、按需加载，而不是高爆炸半径重构

4. **API 收束采用小步推进**
   - 一次只收 1~4 个低风险文件
   - 改完立刻 test/build
   - 红灯就回滚，不保留中间态

5. **项目记忆必须落文件，不靠聊天历史**
   - 这也是新增本文件的原因
