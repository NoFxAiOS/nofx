# NOFX 项目接管总览（中文）

> 状态：初版接管文档
> 时间：2026-03-21
> 目标：为后续深度开发、优化、测试、验收建立统一认知入口。

## 1. 项目定位

NOFX / nofxmax 是一个面向加密货币（并扩展到股票/外汇/贵金属）的 AI 自动交易系统，核心价值在于：

- 多模型接入（OpenAI / Claude / Gemini / DeepSeek / Qwen / Grok / Kimi 等）
- 多交易所接入（Binance / Bybit / OKX / Gate / KuCoin / Hyperliquid / Aster / Lighter 等）
- 自动交易执行（策略 → AI 决策 → 下单/持仓管理）
- 图形化前端工作台（配置、仪表盘、策略工作室、比赛榜单）
- Telegram Agent / x402 支付 / 加密配置能力

## 2. 当前接管目标

本阶段不直接做大规模业务改造，而是先完成：

1. 项目结构扫描
2. 中文架构文档与模块索引整理
3. 运行/测试基线确认
4. 风险点识别
5. 开发治理文件建立

## 4. 第一轮关注重点

按用户优先级：

1. 收益
2. 稳定性

但工程推进顺序建议为：

1. 稳定性基线
2. 风控与交易链路正确性
3. 观测性与测试覆盖
4. 再做收益优化（策略、模型、参数、市场选择）

原因：没有稳定性和一致性，收益评估不可信。

## 5. 下一阶段计划

### Phase 1：认知建模
- 梳理后端主调用链
- 梳理策略生成与 AI 决策链
- 梳理交易执行与持仓同步链
- 梳理前端页面与 API 对应关系

### Phase 2：开发治理
- 完善 TODO / DEVLOG / DECISIONS / TEST_PLAN / ACCEPTANCE
- 建立变更影响分析机制
- 建立模块级中文索引

### Phase 3：核心链路审查
- 决策链：策略 → prompt → 模型调用 → 解析 → 决策落库
- 交易链：决策 → 下单 → 同步 → 持仓/订单状态一致性
- 风控链：仓位、杠杆、网格、强平、异常恢复

### Phase 4：首个正式优化任务
- 选择一个可测、可验收、低爆炸半径的目标做打样优化

## 6. 当前已识别的初始风险

1. 交易系统为高风险场景，任何“收益优化”都必须先建立正确性基线
2. API 层开放面较大，后续需专项检查认证、授权、敏感字段暴露
3. 交易所适配器较多，接口一致性和状态同步可能是潜在故障源
4. 前后端文档虽存在，但仍不足以支撑长期接管，需要补全项目级中文说明
5. 环境变量模板中存在命名与代码读取项可能不完全一致的问题，需后续核验

## 7. 推荐协作规则

- 用户负责业务方向与优先级
- Agent 负责方案、执行、拆解、文档、测试推进
- 默认允许内部探索、安装依赖、跑测试、改文档、建分支、提交
- 涉及外部不可逆影响（生产交易、外发消息、公开操作）再单独确认
