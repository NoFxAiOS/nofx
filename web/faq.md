NOFX 用户反馈与痛点深度分析
高频问题 TOP 20
基于Telegram群聊记录分析，以下是用户最常遇到的问题及其出现频率：

1. 币安API调用失败 (出现频率: ★★★★★)
   典型案例：​

Plain Text
用户反馈: "一直调取币安api失败，vpn试过新加坡，日本，香港都不行，ping是正常的"
错误信息: "<apierror> code=-2015, msg=Invalid API-key, IP, or permissions for action"
</apierror>
原因分析：​

IP白名单未设置

VPN节点IP不稳定

API权限配置不正确

使用了统一账户API而非合约账户

2. 前端端口无法访问 (出现频率: ★★★★★)
   典型案例：​

Plain Text
用户反馈: "3000端口还是打不开呢，已经全部放行了"
"前端网页打开是空的"
解决方案：​

需要同时开放服务器提供商和系统防火墙

使用 npx vite --host 0.0.0.0 --port 3000

检查前后端是否都正常运行

3. 保证金不足问题 (出现频率: ★★★★★)
   典型案例：​

Plain Text
用户反馈: "为什么一直提示保证金不够，我有130U"
错误信息: "ETHUSDT open_short 失败: <apierror> code=-2019, msg=Margin is insufficient"
</apierror>
原因分析：​

资金未划转到合约账户

杠杆倍数设置过高（默认20-50倍）

已有持仓占用保证金

计算逻辑未扣除已占用保证金

4. AI学习数据加载失败 (出现频率: ★★★★☆)
   典型案例：​

Plain Text
用户反馈: "⚠️ 加载AI学习数据失败 这个如何解决啊？"
"仪表盘提示加载ai学习数据失败是正常的吗"
原因：​

TA-Lib库未正确安装

历史数据不足

环境配置问题

5. 子账户杠杆限制 (出现频率: ★★★★☆)
   典型案例：​

Plain Text
错误信息: "code=-4421, msg=Subaccounts are restricted from using leverage greater than 5x"
用户反馈: "子账户不能5倍，这个能解吗"
解决方案：​

使用主账户

修改代码中的杠杆倍数设置

调整仓位大小

6. 持仓模式设置失败 (出现频率: ★★★★☆)
   典型案例：​

Plain Text
错误信息: "code=-4061, msg=Order's position side does not match user's setting"
用户反馈: "设置保证金失败怎么回事？"
原因：​

币安设置为双向持仓，代码要求单向

有持仓时无法切换模式

7. Docker部署问题 (出现频率: ★★★★☆)

Plain Text
用户反馈: "docker老是报错，npm ci --only=production=false报错"
"卡在 go build -o nofx 不动了"
常见问题：​

网络连接问题

依赖安装失败

内存不足（需要至少2C2G）
